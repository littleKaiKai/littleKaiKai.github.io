1. 关于子盒子的宽度的问题。

如果子盒子处在标准流当中，那么子盒子的宽度会自动继承父盒子的宽度，根本不要去谈什么用内容去撑开，而是直接继承的。
如果盒子脱标了，浮动或者是绝对定位了，那么如果不给盒子设置width：100%的话，这个时候，子盒子的宽度会由内容去撑开
行内元素或者是行内块元素的宽度都是由内容撑开的

2. 关于对象判断是否相等的问题

  • 要注意对象只能是等于它自身，其他任何的对象，它都不会去相等。即便是内容什么的相等，== 也是成立不了的

    3.关于通过实例修改原型属性，会不会影响之后new出来的新实例的问题

  • 不管是new出来的实例，还是字面量定义的实例，修改它的原型对象的话，都会影响到后面再new出来的对象的。比如说下面的代码 
    function Student() {
        this.coding = function(){
            console.log(1)
        }
    }
    var stu1 = new Student();
    //stu1.coding();
    stu1.__proto__.say = function(){
        console.log(3)
    }
    //stu1.say();
    var stu2 = new Student();
    stu2.say(); 这里调用say方法的话，是会打印出3的
</script>

4.alert（）的小括号里接收的是字符串参数，所以，如果传入的不是字符串的话，会调用这个参数自身的toString（）方法，根据自身同String（）的规则转换成字符串
