Aug 22, 2016点击

页面中创建元素

作为前端程序员，我们经常会碰到动态在页面中添加一个或者多个htmlElement，在这个时候我们通常的思路可以分为两大方面。

第一种方法是通过html字符串的形式

也就是说，我们首先把我们需要的html字符串写出来，然后再通过html或者innerHtml的方法渲染到页面中。这里面我们要注意的东西有两点：

  • 第一点是，我们在写我们需要的字符串的时候，我们可以手动地拼接，也可以使用模板字符串，具体的实现方法有多种：

    • 第一种是直接手动拼接
    • 第二种是我们可以使用ES6的拼接字符串的语法，这样要比直接拼接快速多了
    • 第三种是，我们可以通过模板引擎的方式来得到我们想要的字符串，这在我们处理大量渲染的时候，是非常高效的。
    • 第四种是，现将字符串保存到数组中，然后调用数组的join方法

  • 第二点是，我们还要注意，在渲染的时候，我们要先拼接好所有的字符串一次性的添加到页面中，因为一次性将所有的元素添加到页面中去的效率，要比多次添加的效率高很多，使用innerHTML和creatElement创建元素的效率，几乎效率一样

    第二种是通过document.creatElement的方式

    这种方法的话，我们同样是需要注意，我们的html字符串首先是需要一个元素来包裹起来，但是有时候我们又不希望多加这个元素，所以，好多时候，我们都会放弃这种方法，来选择第一种方法。但是拼接字符串又确实是每个前端程序员都不喜欢的。这个时候，我们其实还有一个API可以用，这就是document.creatDocumentFragment()

    • 要注意：appendChild()括号中不能是元素集合，只能是一个节点
    • var container = document.creatDocumentFragment()使用这个创建出来的容器，只做为容器，不占用结构，在往页面添加的时候，之后把这个容器中的子元素添加到页面中，而不会把这个容器也添加到页面中
    • var container = document.creatDocumentFragment() 创建出来的元素没有innerHTML的属性
