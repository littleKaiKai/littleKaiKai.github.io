DOM API 查漏补缺

Created at 2016-11-22

有些API啊总是用了忘，用了忘。好记性不如烂笔头，赶紧记下来把，
就当是做一个工具箱了。

  1. document.documentElment这个API获取到的就是我们html标签
  2. getBoundingClientRect()这个API里包含了我们选中元素距离浏览器窗口的上下
    左右距离，并且，还有一个width属性，这个属性是当前元素的宽度
  3. document.documentElement.getBoundingClientRect().width可以获取到html标签
    的宽度，要注意这个属性在移动端获取到的其实是viewport这个视图容器的宽度，
    所以我们在移动端的时候，一定要在meta标签里设置viewport为device-width    这在用rem布局的时候，会用到
  4. document.body.clientWidth，同样的，这个API也可以在rem布局的时候使用，得到
    的是body标签的宽度
  5. window.addEventListener('pageshow',function(){})这个事件是页面显示事件。
    这个事件和window.onload事件差不多，但是在某些浏览器中，有”往返缓存”的特性，
    所以在单击前进后退按钮的时候，有可能不会触发onload事件，所以这个时候我们就需要使用pageshow事件了。
  6. event.persisted事件对象上的这个属性用来判断，在有”往返缓存”特性的浏览器中
    当前页面是从服务器请求的，还是bfcache的
  7. 其实手机上的分辨率的提高，是能够将css中1px的内容展示得更清楚，并不是能够展示更多的东西
